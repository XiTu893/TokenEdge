# TokenEdge PC 端测试聊天功能

## 💬 内置测试聊天

PC 端现在内置了一个简洁的聊天界面，用于快速验证服务有效性！

---

## 🚀 使用步骤

### 1. 启动应用

```bash
npm install
npm start
```

### 2. 选择并下载模型

- 在模型列表中选择一个模型
- 点击"📥 下载"按钮
- 等待下载完成

### 3. 启动服务

- 点击"▶️ 启动"按钮
- 服务启动成功后，API 地址区域会显示
- "💬 打开聊天"按钮变为可用

### 4. 打开测试聊天

- 点击"💬 打开聊天"按钮
- 应用内会打开测试聊天界面
- **无需离开应用即可测试**

### 5. 开始对话

- 在输入框中输入消息
- 按回车或点击"发送"按钮
- 等待 AI 回复
- 验证服务是否正常工作

---

## 🎯 测试聊天界面特点

### 实时状态显示

| 状态 | 显示 |
|------|------|
| ✅ 服务在线 | 绿色圆点 + "服务在线" |
| ❌ 服务离线 | 红色圆点 + "服务离线" |

### API 地址显示

- 顶部显示当前服务地址
- 格式：`http://localhost:3000`
- 端口号自动获取

### 消息气泡

- **用户消息**：蓝色气泡，右侧对齐
- **AI 消息**：灰色气泡，左侧对齐
- **错误消息**：红色边框，显示错误信息

### 快捷操作

- ✅ 回车发送消息
- ✅ 发送时禁用输入框
- ✅ 自动滚动到底部
- ✅ 加载动画显示

## 🔧 故障排查

### 问题 1：点击"打开聊天"没反应

**原因**：服务未启动
**解决**：
1. 确认模型已下载
2. 点击"▶️ 启动"按钮
3. 等待服务启动成功

---

### 问题 2：发送消息后没有回复

**可能原因**：
1. 模型未正确加载
2. 服务端口被占用
3. 模型文件损坏

**解决方法**：
1. 检查服务状态（绿色圆点）
2. 重启服务
3. 重新下载模型

---

### 问题 3：显示"服务离线"

**原因**：API 服务未运行
**解决**：
1. 停止当前服务
2. 重新启动服务
3. 如果仍然失败，检查端口占用情况

## 📝 API 测试

测试聊天界面使用以下 API：

### 聊天补全接口

```http
POST /v1/chat/completions
Content-Type: application/json

{
  "model": "gemma-4",
  "messages": [
    {
      "role": "user",
      "content": "你好"
    }
  ],
  "max_tokens": 512,
  "temperature": 0.7
}
```

### 健康检查接口

```http
GET /health

响应：
{
  "status": "ok",
  "model": "模型路径",
  "port": 3000,
  "modelLoaded": true
}
```

## 💡 提示

1. **测试聊天是内置的**：不需要打开浏览器
